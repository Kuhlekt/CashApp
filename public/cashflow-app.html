Data Sent to AI
⚠ Bank ref, payer name, amount, invoice numbers. No full account or PII sent.
AI Governance
All AI decisions are audit-logged with confidence score, method, and session ID. Human approval required above threshold.
💡 AI suggestions are advisory only. All allocations require human confirmation and are fully audit-trailed. The system never posts to ERP without explicit approval.
⚠ Current Storage Mode: Browser Memory + localStorage
Data is stored in browser memory during your session. Configuration (governance rules, reason codes, on-account items) is persisted to localStorage. All bank/invoice/audit data is session-only and resets on page reload.

🔒 Security Checklist
✓ All API tokens stored in memory only — not in DOM or URL
✓ SHA-256 hash verification on every bank file import
✓ Tamper-evident audit chain with rolling hash
✓ No data transmitted externally except via explicit API calls
⚠ Add HTTPS — never deploy over HTTP
⚠ Add server-side auth (OAuth2/SSO) before production
⚠ Rotate Anthropic API key monthly
✕ Do not commit API keys to git repositories
✕ Do not use browser localStorage for sensitive financial data in production
🏗 Production Architecture
Next.js 14 frontend → Neon PostgreSQL → Kuhlekt API → Anthropic Claude API
Auth: Clerk / NextAuth with SSO
Deploy: Vercel (frontend) + Railway (backend API)
File store: AWS S3 or Cloudflare R2
Monitoring: Sentry + Vercel Analytics
📋 Making It Available
Internal demo: Host on Vercel with password protection (Vercel Pro) or Netlify with basic auth
Client demo: Share the HTML file directly — self-contained, no server needed
Investor demo: Deploy to a clean URL (e.g. demo.cashflow.ai) with a fixed dataset pre-loaded
Production: Full Next.js build with tenant auth, database, and API backend
